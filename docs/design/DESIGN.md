## 1. ข้อตกลงที่แช่แข็ง (ห้ามละเมิดโดยไม่ถามเจ้าของ)

> **⚠️ อ่านตรงนี้ก่อน — ขอบเขตหมายถึงอะไรจริง ๆ** (เจ้าของแก้ความเข้าใจผิดของผม 2026-07-17)
>
> **หน้าตาเปลี่ยนได้เต็มที่** — นั่นคือทั้งหมดของงานนี้ รวมถึงรื้อ markup, จัดเลย์เอาต์ใหม่, เปลี่ยนโครงสร้าง element
>
> **สิ่งที่ห้ามแตะคือ codebase/ฟีเจอร์** — logic, การคำนวณ, flow การทำงาน สิ่งที่เจ้าของไม่อยากต้องกลับมาแก้ซ้ำ
>
> **ถ้าการเปลี่ยนหน้าตาต้องเปลี่ยน "ข้อมูลที่เรียกมาแสดง" → ต้องถามเจ้าของก่อนตัดสินใจ** ห้ามแก้ query เอง
>
> รอบแรกผมตีความว่า "ห้ามแก้โค้ด" คลุมถึง markup ด้วย เลยระวังตัวเกินเหตุ (เช่นกอด props ที่ตายแล้วไว้ · ลังเลจะจัดเลย์เอาต์ใหม่) — **ผิด**

| กฎ | เหตุผล |
|---|---|
| **ขอบเขต = หน้าสาธารณะเท่านั้น** | dashboard admin/designer อยู่บนดีไซน์เก่า และต้องอยู่ต่อจนกว่าจะมีรอบของมันเอง |
| **เปลี่ยนข้อมูลที่เรียกมาแสดง = ต้องถามก่อน** | นี่คือเส้นแบ่งจริงระหว่าง "ดีไซน์" กับ "ฟีเจอร์" |

## 2. Typography

### 2.1 ตระกูลฟอนต์

| บทบาท | ฟอนต์ | ใช้กับ |
|---|---|---|
| **Heading** | Noto Sans Thai | หัวข้อทุกระดับ, ชื่อฟอนต์บนการ์ด, ราคา |
| **Body** | **Noto Sans Thai Looped** | เนื้อความ, คำอธิบาย, ป้ายกำกับ |
| **UI** | Noto Sans Thai | ปุ่ม, nav, control, input |

- โหลดผ่าน **variable font ทั้งคู่** (ละ `weight` ใน `next/font/google`)

> **⚠️ แก้คำอ้างที่ผิด — วัดจริงตอนลงมือ Phase 2**
>
> แผนเดิมเขียนว่า *"เปลี่ยนไป variable แล้วเพิ่มฟอนต์ที่สองยังเบากว่าเดิม"* — **ผิด**
>
> `next/font` **resolve ไป variable font ให้อยู่แล้วตั้งแต่แรก** แม้โค้ดเดิมจะประกาศ 9 static weights ไว้ก็ตาม → build ทั้งสองแบบได้ไฟล์ขนาด **26/15/29 KB เท่ากันเป๊ะ** แปลว่า**ไม่เคยมีความสิ้นเปลืองให้ประหยัด** สมมติฐาน "โหลด 9 weights ทิ้งไว้" ผิดตั้งแต่ต้น
>
> | | ขนาด | ไฟล์ |
> |---|---|---|
> | ก่อน — Noto Sans Thai อย่างเดียว | **71 KB** | 3 |
> | หลัง — + Noto Sans Thai Looped | **140 KB** | 6 |
> | | **+68 KB (เท่าตัว)** | |
>
> **นี่คือราคาของการตัดสินใจให้ body เป็น Looped** — เป็น woff2, cache ได้, `display: swap` การละ `weight` ยังถูกต้อง (โค้ดตรงกับสิ่งที่เกิดขึ้นจริง) แต่**ไม่ได้ให้ส่วนลด**
>
> ถ้าวันหน้าอยากลด: ตัด subset `latin`/`latin-ext` ของ Looped ทิ้งได้ (เนื้อความไทยไม่ต้องใช้) — ประหยัดได้ ~41KB แต่ต้องตรวจว่าไม่มีเนื้อความละตินปนอยู่
- CSS var: `--font-noto-thai` (ชื่อเดิม คงไว้) + `--font-noto-thai-looped` (ใหม่)
- **`Noto Sans Thai UI` ตัดทิ้ง** — ไม่มีใน allowlist ของ `next/font/google` (Google Fonts API เสิร์ฟได้ แต่ next/font ไม่รู้จัก) UI text ใช้ Noto Sans Thai แทน

### 2.2 น้ำหนัก

- **Heading ใช้ 700 (bold) กับ 800 (extra bold) เท่านั้น**
- **ลำดับความสำคัญมาจาก*ขนาด* ไม่ใช่น้ำหนัก** — ห้ามใช้ weight ไล่ชั้น
- ⚠️ ทั้งแอปตอนนี้ใช้แค่ 600/500/400 — **ไม่มี 700/800 ที่ไหนเลย** น้ำหนักหัวข้อคือของใหม่จริง

### 2.3 Type scale — ตรงจาก Figma Styles panel

line-height ใน Figma เป็น `Auto` ทุกตัว = ใช้ metrics ของฟอนต์เอง

**⚠️ แก้ความเข้าใจผิด (ตรวจจริงตอน Phase 3):** เอกสารเคยเขียนว่า "ไม่กำหนด `lineHeight` ในโค้ด = ได้ Auto" — **ผิด**

**Tailwind preflight ตั้ง `html, :host { line-height: 1.5 }`** → ทุกอย่างสืบทอด 1.5 มา การละ `lineHeight` ใน `fontSize` token ไม่ได้ให้ font metrics แต่ได้ **1.5** ยืนยันจาก `getComputedStyle`: `body` 16px → lh 24px · `hero` 84px → lh 126px ทั้งคู่ = 1.5 เป๊ะ

**ยังตรงกับที่เจ้าของสั่ง** ("line-height ใช้เท่าเดิมไปก่อน") เพราะ **1.5 คือสิ่งที่เว็บทำอยู่แล้ววันนี้** — ไม่ได้เปลี่ยนอะไร

**ผลที่ตามมาที่ต้องรู้:** `hero` 96px จะได้ line-height 144px ส่วน Figma แบบ Auto จะได้ราว 130px → **หัวข้อใหญ่ในเว็บจะหลวมกว่าใน moodboard** ถ้าตอนทำ Phase 7 แล้วเห็นว่าโปร่งเกิน แก้ได้ที่ `fontSize` token (ใส่ `lineHeight` ให้เฉพาะ step `hero`/`font-slug`/`h1`) — กระทบเฉพาะหัวข้อ ไม่ลามไปเนื้อความ

**ที่มา: สเปกเต็มทีละสไตล์ใน [`moodboard/style/`](./moodboard/style/)** — อ่านจาก Figma Edit-text-style dialog ตรง ๆ ไม่ได้เดา

| ชื่อ | ฟอนต์ | น้ำหนัก | ขนาด | tracking | ใช้ที่ |
|---|---|---|---|---|---|
| `hero` | Noto Sans Thai | **ExtraBold 800** | **60** | 0% | ข้อความ hero หน้าแรก |
| `font-slug` | Noto Sans Thai | **ExtraBold 800** | **48** | 0% | ชื่อฟอนต์บนหน้ารายละเอียด |
| `h1` | Noto Sans Thai | **ExtraBold 800** | **40** | 0% | หัวข้อหมวด ("ฟอนต์ล่าสุด") |
| `h2` | Noto Sans Thai | **ExtraBold 800** | **24** | 0% | หัวข้อรอง |
| `body` | **Noto Looped Thai** | Regular 400 | **16** | 0% | เนื้อความ |
| `body-sm` | **Noto Looped Thai** | **Light 300** | **14** | 0% | ป้ายกำกับ, ลิงก์ footer |
| `ui` | Noto Sans Thai | **Bold 700** | **16** | 0% | ปุ่ม, nav, control |
| `fc-heading` | Noto Sans Thai | Bold 700 | **16** | 0% | **ชื่อฟอนต์บนการ์ด** |

**หัวข้อทุกระดับเป็น ExtraBold 800 เท่ากันหมด — ลำดับความสำคัญมาจาก*ขนาด*ล้วน** ตรงกับที่เจ้าของสั่งตั้งแต่แรก

**tracking = 0% ทุกตัว — ห้ามใส่ letter-spacing เอง**

> **⚠️ แก้ความผิดพลาดของผมเอง (2026-07-17)** — รอบแรกอ่านจาก Styles panel ภาพเดียวแล้วเดาส่วนที่เหลือ ผิด 5 จุด:
> - `h1` / `h2` ใส่ **700** ทั้งที่เป็น **800** (ไปใส่ weight ไล่ชั้นเอง ซึ่งขัดกับกฎ "ลำดับด้วยขนาด")
> - `body-sm` ใส่ **400** ทั้งที่เป็น **Light 300** ← เจ้าของจับได้จากฟอนต์ลิงก์ใน footer
> - `ui` ใส่ **500** ทั้งที่เป็น **Bold 700**
> - **กุ letter-spacing ขึ้นมาเองทั้งชุด** (-0.01 ถึง -0.03em) ทั้งที่ Figma เป็น **0% ทุกตัว**
> - อ่าน **"FC Heading" เป็น "FG Heading"** แล้วตีความว่า "Footer Group Heading" — ที่จริงคือ **Font Card Heading**
>
> บทเรียน: อย่าอ่านสเปกจากภาพรวมภาพเดียวแล้วเติมช่องว่างด้วยการเดา

### 2.6 สไตล์ที่เพิ่มนอกเหนือจาก Figma

เจ้าของอนุมัติ (2026-07-17) — Figma ไม่มีสองอันนี้ แต่ของจริงต้องใช้:

| ชื่อ | ฟอนต์ | น้ำหนัก | ขนาด | ใช้ที่ |
|---|---|---|---|---|
| `badge` | Noto Sans Thai | Bold 700 | **12** | ป้าย Sale / FREE / NEW / tag |
| `footnote` | **Noto Looped Thai** | Light 300 | **12** | `© 2012–2026 …` ท้าย footer |

แยกสองอันเพราะหน้าที่ต่างกัน — ป้ายต้องหนาแน่นสะดุดตา ส่วน `©` ต้องเงียบ ถ้าใช้สไตล์เดียวกันจะได้อันหนึ่งผิดเสมอ

### 2.4 พื้นล่างของขนาด

**กฎนี้คุม "ข้อความที่ต้องอ่าน" ไม่ได้คุมทุกอย่าง** (แก้จากรอบแรกที่เขียนห้ามลอย ๆ จนตัวเองต้องแอบแหกกฎตอนทำ Badge)

| ประเภท | พื้นล่าง | สไตล์ที่ใช้ได้ |
|---|---|---|
| **เนื้อความ, ลิงก์, ป้ายกำกับที่ต้องอ่านเป็นประโยค** | **14px** | `body` (16) · `body-sm` (14) |
| **ป้ายสั้น ๆ ที่อ่านเป็นก้อน / ข้อความรองท้ายหน้า** | **12px** | `badge` (12) · `footnote` (12) |
| **ต่ำกว่า 12px** | ❌ **ห้าม** | — |

- ปัจจุบันมี `9px`/`10px`/`11px` อยู่ **16 จุด** และ `12px`/`13px` อีก **63 จุด** — `9-11px` **ต้องหายทั้งหมด** ส่วน `12-13px` ที่เป็นเนื้อความต้องขึ้นเป็น 14+
- เหตุผล: อักษรไทยมีสระบน-ล่าง ที่ 9-11px มันตีกันจนอ่านไม่ออก — **ความ "ดูอึดอัด" ของเว็บตอนนี้มาจากตรงนี้พอ ๆ กับเรื่องสี**
- `badge` 12px ยอมได้เพราะเป็นคำสั้น ๆ (SALE / FREE / ชื่อหมวด) ที่จำรูปทรงมากกว่าอ่าน — **เจ้าของอนุมัติ 10-12px แล้ว**

### 2.5 Responsive type

`hero` 96 กับ `font-slug` 48 บนจอ 375px **ล้นแน่นอน** — ทุก step ตั้งแต่ `h1` ขึ้นไปต้องมีค่าที่มือถือ:

ใช้ **`clamp()` ใน `fontSize` token** → responsive อัตโนมัติ **ไม่ต้องเขียน `md:text-*` เลยสักที่** ขนาดไล่ต่อเนื่องตามความกว้างจอ ไม่กระโดดที่ breakpoint

ค่าที่**วัดได้จริง**จาก `getComputedStyle` (ไม่ใช่ค่าที่ตั้งใจ):

| ชื่อ | 375 | 768 | 1280+ |
|---|---|---|---|
| `hero` | 32 | 44 | 60 |
| `font-slug` | 28 | 37 | 48 |
| `h1` | 24 | 31 | 40 |
| `h2` | 20 | 22 | 24 |
| `body` / `body-sm` / `ui` / `fg-heading` | เท่าเดิมทุกจอ | | |

## 3. Color

### 3.1 Palette — 5 ตัวตาม Figma

**เจตนา: ใช้ชื่อตาม Figma ไม่แปลงเป็น role name**

> หมายเหตุ: ตอนวางแผนเคยเสนอให้เลิกใช้ชื่อสี (`navy`/`mint`) แล้วใช้ชื่อตามบทบาท (`accent`/`surface`) — **กลับคำ** เพราะ (ก) palette มีแค่ 5 สี ระบบเล็กเกินกว่าจะคุ้มกับการมีสองคำศัพท์ (ข) Figma เป็นแหล่งความจริงด้านดีไซน์ การให้นักออกแบบกับโค้ดพูดคนละภาษาจะทำให้สองฝั่งเลื่อนออกจากกัน (ค) `navy`/`mint` **มีอยู่แล้วในโค้ดด้วยค่าเดิมเป๊ะ** → ไม่ต้องแตะอะไรเลย admin ปลอดภัยฟรี

**✅ อ่านจาก Figma Edit-color-style dialog ทีละตัวแล้ว** — [`moodboard/style/color_*.png`](./moodboard/style/)

| Figma เรียก | ค่าจริง | ชื่อใน Tailwind | ใช้ที่ |
|---|---|---|---|
| `navy` | `#2B1B3D` | `navy` | **ยังไม่มีที่ใช้บนหน้าสาธารณะ** — เก็บไว้ในระบบเผื่อได้ใช้ (เจ้าของยืนยัน 2026-07-17: หัวข้อทั้งหมดเป็น `black`) |
| `mint` | `#5ECEC8` | `mint` | accent, hover ปุ่ม nav, logo, หัวใจ · "main color mint" |
| **`white`** | **`#F8F8F8`** | **`surface`** ⚠️ | **พื้นการ์ด, แถบล่าง footer, ช่องกรอก, ปุ่มรอง** |
| `black` | `#080808` | `black` | **พื้น nav, พื้น footer**, ตัวหนังสือเข้ม, ราคา |
| `grey` | `#808080` | `grey` | placeholder cover, **ตัวหนังสือรองบนพื้นดำ** (5.07:1 ✅) |

> **⚠️ `white` ของ Figma = `#F8F8F8` ไม่ใช่ `#FFFFFF`** — ในโค้ดจึงตั้งชื่อว่า **`surface`** ไม่ใช่ `white`
>
> เพราะ `white` เป็น token ของ Tailwind อยู่แล้ว การทับมันจะกระทบ **`bg-white` 156 จุด + `text-white` 71 จุด กระจายใน admin/designer 18 ไฟล์** → พาเนล admin จะกลายเป็นเทาอ่อนลอยบน `body` ที่ hardcode `#fff` ไว้ = **เห็นได้ชัด** ต่างจากกรณี `black` ที่ยอมทับเพราะเป็น overlay โปร่งแสงทับรูป (§9.1)
>

## 4. รูปทรง (Shape)

### 4.0 🚫 ไม่มีเส้นกรอบ ไม่มีเส้นขอบ — ทุกส่วนของงาน

**เจ้าของสั่งตรง ๆ (2026-07-17): "เราไม่เอาเส้นกรอบ เส้นขอบ ในทุกส่วนของงานดีไซน์"**

**แยกลำดับชั้นด้วย "พื้นสี" แทนเส้น:**

| แทนที่จะใช้ | ให้ใช้ |
|---|---|
| การ์ดมีกรอบ | พื้น `surface #F8F8F8` ลอยบน `body #FFFFFF` |
| ปุ่ม outline (กรอบดำ) | พื้น `surface` |
| ช่องกรอกมีกรอบ | พื้น `surface` (บน nav ดำยิ่งเด่นเอง — moodboard/nav bar.png) |
| ป้าย tag มีกรอบ | พื้น `surface` |
| เส้นคั่นแถบหัว modal | พื้นต่างระดับ |
| เส้นคั่น footer | ไม่ต้องมี — พื้นดำกับพื้น surface แยกกันเองอยู่แล้ว |

**ข้อยกเว้นเดียว: `focus-visible`** ใช้ CSS `outline` (ไม่ใช่ `border`) โผล่เฉพาะตอนกด Tab — เป็นเรื่อง **a11y ไม่ใช่การตกแต่ง** ถ้าตัดทิ้งจะไม่มีใครรู้ว่ากำลังโฟกัสอยู่ที่ไหน

### 4.1 มุม — เหลี่ยม

**ไม่มีมุมมน** ตามที่เจ้าของสั่ง ("ต่อไปเราจะไม่มีมุมมน มันไม่เท่ / เราจะใช้สี่เหลี่ยมเลย")

- **การ์ด, ปุ่ม, อินพุต, พาเนล, ป้าย Sale/FREE/NEW, tag → เหลี่ยมหมด**
- **ของที่กลมจริง ๆ ยังกลม** (`rounded-full`): avatar ใน Nav, ปุ่มไอคอนวงกลม (หัวใจ), จุดบอกตำแหน่ง carousel
- **ลูกศร carousel ใน moodboard ไม่มีวงกลมรองพื้นแล้ว** — เป็นลูกศรเปล่า ๆ บนภาพ (ของเดิมมีพื้นวงกลม `bg-black/20`)

**วิธีทำ: ไม่ต้องแตะ `borderRadius` ใน config เลย** — `border-radius` ใน CSS มีค่าเริ่มต้นเป็น `0` อยู่แล้ว และ Tailwind preflight ไม่ได้ใส่ให้ → **แค่ไม่ใส่ class `rounded-*` ก็เหลี่ยมแล้ว**

> ❌ **ห้ามตั้ง `borderRadius.DEFAULT: 0`** — ตรวจแล้วมี `rounded` เปล่า ๆ อยู่ **8 จุด และอยู่ใน admin/designer ทั้งหมด** (`admin/page.tsx:129-130`, `admin/font-review/page.tsx:282-283`, `designer/(dashboard)/page.tsx:125-126`, `admin/FontForm.tsx:581,583`) หน้าสาธารณะไม่มีเลยสักจุด → ทับ `DEFAULT` = thumbnail กับป้ายใน admin เหลี่ยมตาม = **ผิดขอบเขต**

ของเดิมมี radius **13 แบบ** ปนกัน (`rounded-[9px]`, `rounded-[6px]`, `rounded-xl`, `rounded-2xl`, `rounded-[14px]`, …) → **ยุบเหลือ 2: ไม่ใส่อะไรเลย (เหลี่ยม) กับ `rounded-full` (กลม)**

### 4.2 Spacing

- ยึด scale ของ Tailwind (ฐาน 4px) — **ห้ามใช้ arbitrary spacing**
- ของเดิมมี arbitrary value **474 จุด** ใน 8 ไฟล์ในขอบเขต ตัวเลขนี้ต้อง**ลดลงทุก phase ไม่ใช่เพิ่ม**

### 4.3 Elevation

เหลี่ยม + เงา ต้องคุมมือ ไม่งั้นดูเป็นกล่องลอย — moodboard แทบไม่ใช้เงาเลย ใช้**เส้นคั่น**กับ**พื้นต่างระดับ**แทน

|---|---|---|
| `shadow-sm` | `0 1px 2px rgba(0,0,0,0.04)` | ยกเบา ๆ |
| `shadow-md` | `0 2px 12px rgba(0,0,0,0.08)` | hover การ์ด (ค่าเดิมของ FontCard) |
| `shadow-lg` | `0 8px 32px rgba(0,0,0,0.12)` | modal |

| `duration-base` | `250ms` |

## 5. Responsive

### 5.1 หลักการ

**Mobile-first เด็ดขาด** — class เปล่า = มือถือ แล้วค่อยเพิ่ม `sm:`/`md:`/`lg:` ขึ้นไป

ตรงข้ามกับของเดิมที่เขียนเดสก์ท็อปก่อนแล้วแปะ breakpoint ทีหลัง ซึ่งเป็นสาเหตุที่ทั้งแอปมี breakpoint แค่ **53 จุด**, `lg:` ใช้ **ครั้งเดียว**, `xl:`/`2xl:` **ศูนย์** และ `FontCard.tsx` — การ์ดที่ซ้ำอยู่ 5 พื้นผิว — **ไม่มี breakpoint สักตัว**

### 5.2 Breakpoint — ใช้ค่าเริ่มต้นของ Tailwind ไม่ override

| ชื่อ | ค่า |
|---|---|
| `sm` | 640px |
| `md` | 768px |
| `lg` | 1024px |
| `xl` | 1280px |
| `2xl` | 1536px |

> **⚠️ บทเรียนราคาแพง (พบตอน Phase 5)** — เคยตั้ง `screens: { sm: 375, md: 768, lg: 1280, xl: 1536 }` ไว้ใน Phase 2 แล้วเคลมว่า "เพิ่มอย่างเดียว ไม่เปลี่ยนหน้าตา" **ผิดทั้งคู่**
>
> **การเปลี่ยนนิยาม breakpoint ไม่ใช่การเพิ่ม — มันเปลี่ยนความหมายของ `sm:` ทุกตัวที่เขียนไว้แล้ว** (13 จุดใน 7 ไฟล์ รวม `/quote`, `/become-a-designer` ที่ยังไม่ถึงคิว)
>
> ผลจริงที่วัดได้: `grid-cols-2 sm:grid-cols-3 md:grid-cols-4` ที่จอ **375px กลายเป็น 3 คอลัมน์ กว้างคอลัมน์ละ 95.66px** (เดิม 2 คอลัมน์ ~150px) การ์ดแคบจนใช้ไม่ได้
>
> **ต้นตอความสับสน: `375` ไม่ใช่ breakpoint** — ใน mobile-first มันคือ **ฐาน** (เขียน class เปล่า ๆ) ส่วน breakpoint คือจุดที่ layout เปลี่ยน*เหนือ*ฐาน · **375 / 768 / 1280 ใน §5.4 คือ "ความกว้างที่ใช้ตรวจงาน" คนละเรื่องกับ breakpoint**

### 5.3 Container

- `max-w-site` = 1200px **คงเดิม**
- **padding ต้องไล่ตามจอ** — ของเดิมใช้ `px-8` ตายตัวทุกจอ ทำให้มือถือ 375px เหลือพื้นที่จริงแค่ 311px

`ui/Container` = `max-w-site mx-auto px-4 md:px-6 lg:px-8`

| จอ | padding | ที่ 375px เหลือพื้นที่ |
|---|---|---|
| ฐาน (< 768) | `px-4` = 16px | **343px** (เดิม 311px) |
| `md` 768+ | `px-6` = 24px | |
| `lg` 1024+ | `px-8` = 32px | |

### 5.4 ความกว้างที่ใช้ตรวจงาน (ไม่ใช่ breakpoint)

**ทุก component ต้องระบุพฤติกรรมที่ 375 / 768 / 1280** — component ที่ยังไม่ระบุ **ถือว่ายังไม่เสร็จ**

### 🐛 บั๊กของเดิมที่เจอระหว่างทาง — carousel ตายถาวรถ้าโหลดในแท็บที่ซ่อนอยู่

**พบตอน Phase 4 · เป็นของเดิม ไม่ได้เกิดจากงานนี้** (`page.tsx` เหมือน `v1-pre-redesign` ทุกไบต์)

**อาการ:** เปิดหน้าแรกด้วย cmd+click / "เปิดในแท็บใหม่" / กู้ session → พอสลับกลับมาดู **carousel "คัดสรรพิเศษ" ว่างเปล่าถาวร** กู้ไม่ได้จนกว่าจะรีเฟรช

**สาเหตุ** (`page.tsx:73-91`):
1. `animated` เริ่มที่ `false` แล้วพึ่ง `requestAnimationFrame` ซ้อนสองชั้นเพื่อพลิกกลับเป็น `true`
2. แท็บที่ `document.visibilityState === "hidden"` **ไม่ยิง rAF เลย** → `animated` ค้าง `false` ตลอด
3. `animated === false` → `transition: none` บน strip
4. ไม่มี transition → **`transitionend` ไม่ยิง** → `onTransitionEnd()` ที่ทำหน้าที่ snap-back ไม่ทำงาน
5. แต่ `setInterval` ของ autoplay **ยังเดินต่อ** (browser ไม่ throttle แรงเท่า rAF) → `pos` ไต่เลย `poolSize + showCount` ไปเรื่อย ๆ
6. การ์ดถูก `translateX` ออกนอกจอถาวร

**ยืนยันด้วยการวัด:** ในแท็บ hidden — rAF ยิง **0 ครั้งใน 1.5 วินาที** · `inlineTransition: "none"` · `transitionend` ยิง **0 ครั้งใน 9 วินาที** · `pos` ไล่ 5→7→8→9 ไม่ snap กลับสักครั้ง

**ยังไม่แก้** — เป็น logic ของ carousel ซึ่งอยู่นอกขอบเขต **แต่ Phase 7 จะรื้อส่วนนี้เป็น hero slider อยู่แล้ว และแตะ `showCount` ด้วย → แก้ตรงนั้นทีเดียว** ทางแก้: อย่าพึ่ง rAF เพื่อกู้ `animated` (ใช้ `onTransitionEnd` + guard ที่ไม่ต้องรอ transition ก็ได้) หรือหยุด autoplay เมื่อ `document.hidden`

> **ผลข้างเคียงต่อการตรวจงาน:** screenshot หน้าแรกจาก preview pane จะเห็น carousel ว่าง **นี่คือ artifact ของบั๊กนี้ ไม่ใช่ regression** — เบราว์เซอร์จริงที่แท็บอยู่หน้าสุดจะทำงานปกติ

## 9.1 การตัดสินที่เกิดขึ้นตอนลงมือ Phase 2

สองเรื่องที่ไม่ได้อยู่ในแผน แต่โผล่มาตอนเขียนโค้ดจริง — บันทึกไว้เพราะทั้งคู่กลับคำจากแผนเดิม

### สีอยู่ใน `tailwind.config.ts` ที่เดียว — `globals.css` ไม่ได้แตะเลย

แผนเดิมเขียนว่าให้ `globals.css` เป็นแหล่ง token เดียว แล้ว `tailwind.config.ts` อ้าง `var(--x)` เพื่อกำจัดการประกาศซ้ำสองที่

**ทำแบบนั้นไม่ได้** — Tailwind คำนวณ **alpha modifier** จาก `var()` ที่เก็บ hex ไม่ได้ ถ้าทำ `black: "var(--black)"` แล้ว **`bg-black/50` · `bg-black/20` · `bg-white/20` · `bg-white/35` จะพังทันที** (ทั้งหมดนี้ถูกใช้อยู่จริงใน `FontForm.tsx`, `FontDetail.tsx`, `DesignerDetail.tsx`, `FontCard.tsx`) ทางที่ถูกคือเก็บเป็น channel (`--black: 8 8 8`) + `rgb(var(--black) / <alpha-value>)` ซึ่งเพิ่มความซับซ้อนโดยไม่ได้อะไรกลับมา

**ทางออกที่ใช้:** สีอยู่ใน `tailwind.config.ts` **ที่เดียว** ส่วน `:root` ใน `globals.css` เก็บแค่ฟอนต์ (ซึ่ง `next/font` ฉีดให้เอง) → **กำจัดความซ้ำซ้อนได้เหมือนกัน โดยไม่พัง และไม่ต้องแตะ `globals.css` เลยสักบรรทัด** = ลดไฟล์เสี่ยงลงอีกหนึ่ง

### `black: #080808` ทับ token ของ Tailwind — ข้อยกเว้นที่จงใจ

Tailwind มี `black` = `#000000` อยู่แล้ว ประกาศทับ = กระทบทั้งแอป **รวม admin**

จุดที่ได้รับผลกระทบ:
- `admin/FontForm.tsx:581,583` — `bg-black/50` (ป้ายลำดับบนรูป preview) **นี่คือ admin**
- `FontDetail.tsx:278,287` · `DesignerDetail.tsx:203-204` — `bg-black/20` (สาธารณะ อยู่ในขอบเขตอยู่แล้ว)

**ยอมรับ** เพราะ `#080808` ต่างจาก `#000000` แค่ 3% และทุกจุดที่ใช้เป็น overlay ที่ opacity 20-50% ทับบนรูปที่ผู้ใช้อัปโหลด → **ความต่างต่ำกว่าที่ตาคนแยกออก** แลกกับการรักษาคำศัพท์ให้ตรงกับ Figma ซึ่งกันไม่ให้ดีไซน์กับโค้ดเลื่อนออกจากกัน

## 12. การตรวจงาน

`npm run dev` ผ่าน launch config `Next.js Dev` (port 3000) ทุก page phase:

1. **ก่อน:** checkout tag ฐานของ phase → screenshot ที่ **375 / 768 / 1280** — หน้าที่ของภาพนี้**ไม่ใช่**เทียบพิกเซล แต่คือจับ**การสูญหายเชิงโครงสร้าง** (การ์ดหาย, ปุ่มไม่ render) พิกเซลเลื่อน**คือประเด็นของงานนี้**
2. **Console:** `onlyErrors: true` ต้องว่าง — React key warning กับ hydration error คือโหมดพังที่เกิดจริงเวลา restyle ผลลัพธ์ `.map()`
3. **Network:** query Supabase ต้องยังยิงและยังได้แถวกลับ — **นี่คือการตรวจฟังก์ชันตัวจริง** จับโหมด "401 = กริดฟอนต์ว่างเปล่า" ; เช็คด้วยว่าโหลดฟอนต์ **2 ตระกูล** (Noto Sans Thai + Looped) รวม ~140KB
4. **ตรวจระบบ:**
   - ไม่มี horizontal scroll ที่ 375px — `document.documentElement.scrollWidth > window.innerWidth`
   - arbitrary value **ลดลง ไม่ใช่เพิ่ม** — `grep -oE '\[[^]]+\]' <ไฟล์>`
   - ไม่มี `text-[9|10|11|12|13px]` เหลือในไฟล์ที่แตะแล้ว
   - ไม่มี `rounded-*` นอกจาก `rounded-full`
   - `getComputedStyle` ของเนื้อความ = **Looped** ; **และ admin ต้องยังเป็น Noto Sans Thai** (พิสูจน์ว่าไม่หลุดขอบเขต)
   - **`focus-visible` มองเห็นได้ทุก element ที่กดได้**
5. **Interaction:** หน้าแรก — ลูกศร/จุด/autoplay วนครบสองด้าน (`onTransitionEnd` snap-back คือจุดเปราะ ดูว่ากระตุกไหม) **+ ทั้ง 3 breakpoint หลังแก้ `showCount`** ; `/fonts` — filter, favourite ตอน logout ต้อง redirect พร้อม `?next=` ถูกตัว ; `FontDetail` — lightbox เปิด/ปิด/Esc, TypeTester, **ตรวจราคาครบทั้ง 5 กิ่ง**ด้วยฟอนต์จริงในแต่ละสถานะ ; `Nav` — search ทั้งสองความกว้าง, เมนูตาม auth ทั้ง login/logout
6. **ไม่ตรวจ dark** — ไม่มีอยู่
7. **`npm run build` ก่อน tag ทุก phase** — `output: "export"` แปลว่า static export พัง = deploy พัง และไม่โผล่ใน `next dev`

### 13.2 `CoverCarousel` — component สไลด์ (ใช้ซ้ำได้)

`src/components/CoverCarousel.tsx` — **ยกออกจาก `page.tsx` เพื่อ reuse ที่ `DesignerDetail` (เจ้าของยืนยันจะใช้)**

```tsx

```
- รับ pool ฟอนต์จากภายนอก (component ไม่ query เอง) → หน้าแรกส่ง `buildSliderPool()` · **DesignerDetail จะส่งฟอนต์ของดีไซเนอร์นั้น "แสดงผลเหมือนกัน"**

### 13.6 บทเรียนรอบนี้

- **`text-*` เป็น token ขนาด/น้ำหนัก ไม่ได้ตั้ง font-family** — เนื้อความต้องใส่ **`font-body`** ด้วยถึงจะเป็น Looped (เคยลืมทั้งหน้า จนเนื้อความเป็น Noto Sans Thai หมด) · หัวข้อใส่ `font-heading`
- **`cn()` ไม่ merge tailwind** — class ที่มาทีหลังไม่ชนะเสมอ **อย่าพึ่ง override ผ่าน `className`** ให้เพิ่ม variant/size ใน component แทน
- **`{/* comment */}` วางเป็นบรรทัดแรกของ ternary branch ใน JSX = parse error** (`) : cond ? ( {/* … */} <div>`) ต้องใช้ `// comment` แทน
- **เจ้าของทำงานขนานมาในไฟล์เดียวกัน** (เช่นแก้ Footer เอง) — **stage เฉพาะไฟล์ที่เราแก้** อย่า `git add -A` · `next-env.d.ts` ห้าม commit (flip ทุก build)
- อย่าแก้เกินที่สั่ง — เคยรื้อ layout เพื่อ baseline แล้วทำ gap เสียจนเจ้าของทัก
